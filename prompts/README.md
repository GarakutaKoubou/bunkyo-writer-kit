# 市民ライター向け AIプロンプト バージョン管理

## ファイル命名規則
```
{対象AI}_v{major}.{minor}_{YYYYMMDD}.md
```
- **major**：ルール体系の大幅変更（新ルール追加・削除）時にインクリメント
- **minor**：表現の修正・ルールの追記など小幅な変更時にインクリメント
- **日付**：ファイル作成日（更新日ではなくファイルの新規作成日）

## 対象AIの種別
| ファイル名プレフィックス | 対象 |
|----------------------|------|
| `claude_project_` | Claude（claude.ai）の「プロジェクト」機能 |
| `gemini_gem_` | Google Gemini の「Gem」機能 |

## 更新のタイミング（目安）

- `writing_rules.md` に新ルールが追加されたとき
- 編集長から新しい指摘パターンが蓄積されたとき（`editor_feedback.md` 更新後）
- 市民ライターから「わかりにくい」という声が届いたとき

## 更新手順
1. 既存の最新ファイルをコピーしてバージョン番号と日付を変更
2. 内容を編集
3. このREADMEのバージョン履歴に1行追記
4. 古いバージョンは削除せず残す（差分確認のため）
