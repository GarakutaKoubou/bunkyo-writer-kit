# ファクトチェック・品質チェックルール

> プレビューを表示する前に、`article_check.py` + Claude Code自身によるファクトチェックの**両方**を必ず実施する。

## チェック手順

```bash
# ① article_check.py による自動チェック（執筆ルール・データ整合性）
python3 article_check.py --json /tmp/bunkyo_YYYYMMDD/article.json
# ② エラーがあれば修正してから再チェック
# ③ チェック通過後、Claude Codeがファクトチェックを実施してチャットに表示
# ④ プレビュー生成
python3 preview_generator.py --json /tmp/bunkyo_YYYYMMDD/article.json --open
```

## Claude Codeが実施するファクトチェック（チャットに必ず表示）

article_check.py では検出できない「素材との照合」をClaude Codeが実施する：

```
## ファクトチェック結果（素材との照合）
- 固有名詞（人名・団体名・施設名）：✅ 一致 ／ ⚠️ 「〇〇」→素材では「△△」
- 日付・時間：✅ 一致 ／ ⚠️ 「〇月〇日」→素材では「△月△日」
- 住所：✅ 素材に「〇〇」と明記あり → 一致 ／ ⚠️ 素材に住所の記載なし → **要確認（記事に書かない）**
- 数値（金額・人数・面積など）：✅ 一致
- 素材にない情報の追加：✅ なし ／ ⚠️ 「〇〇」は素材に記載なし
```

### チェック項目の詳細

**ファクトチェック（Claude Codeが実施）：**
1. **固有名詞**：人名・団体名・施設名がすべて素材と文字単位で一致しているか
2. **日付・時間**：すべての日付・時刻が素材と一致しているか（曜日を書いていないか）
3. **住所**（クリティカル・必須）：
   - 素材に住所が明記されていれば、「文京区〇〇N-NN-NN」形式に変換する
   - 素材に住所が**明記されていない**場合：「住所未確認」とチャットに明記し、ユーザーに確認を求める。**推測・記憶・施設名からの類推で書いてはならない**
   - 住所の町名（○○町・○○丁目等）は特に誤りやすい。素材原文の文字と1文字単位で照合する
4. **数値**：金額・人数・面積・回数などがすべて素材と一致しているか
5. **素材にない情報**：素材に書かれていない事実を追加していないか
6. **登録商標®・™**：本文中の®・™記号は除外する。ただし、素材に®が付いている語が含まれる場合は、**本文末尾（最終段落の後）**に以下の注記を追加する：
   ```
   ※「〇〇」は、株式会社△△の登録商標です。
   ```
   - 〇〇：®が付いていた語（例：クレパス）
   - △△：その商標の権利者企業（素材から特定する。不明な場合は「[商標権者名・確認要]」と記す）
   - 複数の登録商標がある場合は複数行に並べる

**執筆ルールチェック（article_check.py が自動実施）：**
- タイトル文字数・日付有無・「開催」有無
- コメント語順（主語→コメント→と話す）
- 語尾「だ」・「語る」使用
- 記号全角統一・漢字変換・カタカナ語
- 禁止表現・括弧ルール

## 記事の深さ・情報量チェックリスト（記事を書く前に確認）

- [ ] 展示・イベントの具体的な展示物・プログラム名を全て書いたか
- [ ] 各展示物の「背景・意義・面白さ」を1文以上添えたか（事実の列挙で終わっていないか）
- [ ] プレスリリースの「主な展示内容」「エリア詳細」を全て読んだか
- [ ] 「単なる数字・名前」だけでなく、読者が「行きたい・体験したい」と思える描写があるか
- [ ] コメントは原文の「ですます」調を「である」調に変換したか

## PDFが読めなかった場合の対処

ReadツールでPDFを読んだ際、本文テキストが取得できなかった場合：

1. **直ちに作業を止める**
2. ユーザーに報告する：「PDFの本文テキストが取得できませんでした。テキストをコピー＆ペーストしていただけますか？」
3. **絶対にやってはいけないこと**：記憶だけで記事を生成する、推測して書く、報告せず続ける
4. 代替手段：`pages: "1-3"` で再試行、ユーザーにテキストを貼り付けてもらう
